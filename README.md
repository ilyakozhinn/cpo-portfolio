# CPO Portfolio Dashboard

Локальный дашборд для CPO: портфель проектов ATOM, еженедельные статусы, загрузка людей и приоритизация.

## Роли и доступ

| Роль | Что видит | Что заполняет |
|------|-----------|---------------|
| **C-level** | Весь портфель, все отделы, asks | Всё + админка |
| **PO** | Только назначенные проекты | Статус отдела «Продукт» |
| **PM** | Только назначенные проекты | Статус отдела «Разработка» |
| **Маркетолог** | Только назначенные проекты | Статус отдела «Маркетинг» |

### Демо-аккаунты (пароль для всех: `atom2026`)

| Роль | Email | Проекты |
|------|-------|---------|
| C-level | `cpo@atom.local` | Весь портфель + админка |
| PO | `po@atom.local` | JGGL App, JGGL Website |
| PM | `pm@atom.local` | JGGL App, JGGL Buddy + devices |
| Маркетолог | `marketer@atom.local` | JGGL Website, JGGL App |

C-level создаёт аккаунты и назначает проекты в разделе **Админка**.

## Telegram-бот

PO / PM / маркетологи могут сдавать weekly status через бота.

1. Пользователь пишет боту `/start` — без роли бот **молчит**, но заявка появляется в админке.
2. C-level в **Админке → Заявки из Telegram** выдаёт роль и назначает проекты.
3. После доступа: `/status` запускает воронку **Шаг 1…7** (проект → RAG → прогресс → задачи → риски → блокеры → ask → milestone).

Команды: `/status`, `/cancel`, `/help`.

## AI-саммари (DeepSeek)

После сдачи статусов отделов DeepSeek:
- пишет краткое саммари проекта для C-level;
- ставит общий RAG (зелёный / жёлтый / красный) по уровню проблем и выполнению плана;
- в боте спрашивает, выполнены ли задачи прошлой недели.

## Быстрый старт (2 минуты)

```bash
cd ~/Projects/cpo-portfolio
npm install
npm run db:migrate
npm run db:seed
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000).

## Что внутри

- **Command Center** — сводка портфеля, RAG, asks, кто не сдал статус
- **Проекты** — полный список с владельцами и доменами
- **Еженедельный статус** — короткая форма отчёта (RAG, прогресс, риски, ask, milestone)
- **Люди и загрузка** — heatmap % allocation по неделям
- **Приоритизация** — порядок active-проектов и стратегический вес
- **Настройки** — выбор отчётной недели, добавление проектов

## Портфель

При первом запуске `db:seed` загружает ваш реальный портфель:

- 24 направления (JGGL — высший приоритет)
- ~40 проектов с владельцами и доменами
- 20 людей
- 4 проекта на паузе

## Еженедельный ритуал

1. В шапке выберите текущую неделю (или нажмите «Сегодня»)
2. Перейдите в **Еженедельный статус** и заполните формы по active-проектам
3. На **Command Center** проверьте asks, риски и «не сдали статус»
4. На **Люди и загрузка** проставьте % allocation

## Полезные команды

```bash
npm run db:reset   # сброс БД и повторный seed
npm run build      # production-сборка
```

npm install

npm run db:seed        # только первый раз
